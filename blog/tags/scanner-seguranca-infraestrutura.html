Checkov: O Scanner de Segurança que Todo Profissional de IaC Deveria Usar​

Se você trabalha com Infraestrutura como Código (IaC) – seja Terraform, CloudFormation, Kubernetes ou ARM Templates –, já deve ter se perguntado:

"Será que essa configuração está realmente segura?"

O Checkov é um scanner de segurança open-source criado pela Bridgecrew (parte da Palo Alto Networks) que analisa automaticamente seu código IaC em busca de vulnerabilidades antes do deploy.

Com mais de 80 milhões de downloads, o Checkov é usado por empresas como AWS, Google e Microsoft para proteger infraestruturas na nuvem.

✅ O que você vai ver neste artigo:​

  • O que é o Checkov e como ele funciona
  • Os 5 erros mais críticos que ele ajuda a evitar
  • Como integrar com CI/CD para segurança contínua
  • Exemplo prático com Terraform + Azure

Por que o Checkov é Essencial para IaC​

1. O Problema: Configurações Inseguras na Nuvem​

Segundo o Relatório de Ameaças na Nuvem 2023 da Palo Alto Networks, 76% das organizações já sofreram incidentes por erro de configuração.

Exemplos comuns:

  • Buckets S3 públicos (vazamento de dados)
  • Storage Accounts no Azure sem restrição de rede
  • Kubernetes com permissões amplas demais

O pior: muitos problemas só são descobertos após o ataque.

2. A Solução: Checkov​

O Checkov é um scanner SAST (Static Application Security Testing) que:

  • 🔍 Analisa seu código IaC (.tf, .yaml, .json)
  • 🚨 Detecta vulnerabilidades e violações de compliance
  • 🔧 Sugere correções e boas práticas
  • 🔄 Integra com GitHub Actions, GitLab CI, Azure DevOps, CircleCI

Baseado em mais de 1.000 regras derivadas de:

  • CIS Benchmarks
  • AWS/Azure/GCP Security Best Practices
  • GDPR, HIPAA, SOC 2

Como o Checkov Funciona​

  1. Análise do Código: escaneia arquivos Terraform, Kubernetes, etc.
  2. Comparação com Regras: verifica se configurações violam boas práticas.
  3. Geração de Relatório: mostra o erro, linha afetada e sugestão de correção.

Exemplo de saída:​

Check: CKV_AZURE_59: "Ensure that Storage accounts disallow public access"
FAILED for resource: azurerm_storage_account.example
File: /main.tf:25-30
Guide: https://docs.bridgecrew.io/docs/ensure-that-storage-accounts-disallow-public-access

🛑 5 Erros Críticos que o Checkov Detecta​

1. Storage Account Pública no Azure​

resource "azurerm_storage_account" "inseguro" {
  name                     = "stexemplo"
  resource_group_name      = azurerm_resource_group.exemplo.name
  location                 = "eastus"
  account_tier             = "Standard"
  account_replication_type = "LRS"
}

Correção sugerida:

network_rules {
  default_action = "Deny"
  ip_rules       = ["SEU_IP"]
}

2. Bucket S3 sem Criptografia​

resource "aws_s3_bucket" "exemplo" {
  bucket = "meu-bucket-inseguro"
}

Correção:

resource "aws_s3_bucket_server_side_encryption_configuration" "exemplo" {
  bucket = aws_s3_bucket.exemplo.id
  rule {
    apply_server_side_encryption_by_default {
      sse_algorithm = "AES256"
    }
  }
}

3. Kubernetes sem resource limits​

resources:
  limits:
    cpu: "500m"
    memory: "512Mi"

4. Azure SQL sem firewall​

resource "azurerm_sql_server" "example" {
  public_network_access_enabled = true
}

5. IAM Overprivileged​

resource "aws_iam_policy" "admin" {
  policy = jsonencode({
    "Effect": "Allow",
    "Action": "*",
    "Resource": "*"
  })
}

Como Usar o Checkov no Seu Projeto​

1. Instalação​

pip install checkov
checkov --version

2. Rodando localmente​

checkov -d /seu/projeto/terraform

3. Integração com GitHub Actions​

- name: Run Checkov
  uses: bridgecrewio/checkov-action@v1
  with:
    directory: .
    fail_on: HIGH

✅ Conclusão: Vale a Pena Usar?​

O Checkov é uma das ferramentas mais poderosas para quem leva segurança em IaC a sério.

Prós:​

  • Open-source e gratuito
  • Suporte a múltiplas clouds
  • Rápido e fácil de usar

Contras:​

  • Pode gerar alguns false positives (resolvido com .checkov.yaml)
  • Integração manual em pipelines mais complexos
