AULA 4.5 - GPT STORE: BUILDING CUSTOM GPTS

Material de Referência para Instrutor​

INTRODUÇÃO​

Pergunta Central: Como criar um assistente de IA personalizado que entenda perfeitamente suas necessidades específicas de DevOps e esteja disponível 24/7 para sua equipe?

GPT Store revolucionou a forma como usamos IA. Ao invés de sempre recriar contexto em cada conversa, agora você pode criar assistentes especializados que já entendem seu ambiente, suas ferramentas e seus padrões de trabalho.

O Problema da Especialização: Todo profissional DevOps trabalha com stack específico, padrões únicos da empresa, e cenários técnicos repetitivos. Explicar contexto toda vez que usa ChatGPT é ineficiente e frustrante.

A Solução dos Custom GPTs: Imaginem ter um assistente que já conhece suas configurações Terraform, entende os padrões da empresa, sabe quais modules vocês usam, e pode gerar código seguindo exatamente seus guidelines.

Por que Custom GPTs são perfeitos para DevOps:

  • Especialização Profunda: Treinado especificamente para suas necessidades
  • Contexto Persistente: Lembra suas preferências e padrões
  • Disponibilidade 24/7: Equipe inteira pode usar o mesmo assistente
  • Evolução Contínua: Melhorar baseado no feedback da equipe
  • Integração Natural: Funciona como membro especializado da equipe

Transformação Real: Ao invés de explicar "use nosso module VPC com tags padrão X, Y, Z" toda vez, seu Custom GPT já sabe disso automaticamente.

O QUE VAMOS APRENDER​

Objetivos de Aprendizagem:​

  1. Entender como Custom GPTs diferem do ChatGPT normal
  2. Criar um GPT personalizado para assistência Terraform
  3. Configurar instruções e knowledge base eficazes
  4. Testar e iterar para máxima eficiência

Competências Desenvolvidas:​

  • Criação de assistentes IA especializados
  • Configuração de knowledge base para contexto técnico
  • Otimização de prompts para casos específicos
  • Gestão e evolução de Custom GPTs

O QUE SÃO CUSTOM GPTS​

Definição Simples:​

Custom GPTs são versões especializadas do ChatGPT que você configura com instruções específicas, knowledge base personalizada, e comportamentos direcionados para casos de uso específicos.

Diferenças do ChatGPT Normal:​

ChatGPT NormalCustom GPT
Contexto zero a cada conversaContexto persistente configurado
Conhecimento geralKnowledge base especializada
Precisa explicar tudo sempreJá entende seu ambiente
Uma conversa por vezCompartilhável com equipe

Componentes Principais:​

1. Instructions (Instruções)​

  • Como o GPT deve se comportar
  • Que tipo de especialista deve ser
  • Padrões e guidelines a seguir

2. Knowledge Base​

  • Documentos específicos da empresa
  • Templates e exemplos
  • Best practices internas

3. Capabilities (Capacidades)​

  • Web browsing para info atualizada
  • DALL-E para diagramas
  • Code Interpreter para análise

LABORATÓRIO PRÁTICO: TERRAFORM ASSISTANT GPT​

Cenário do Lab:​

Vamos criar "TerraformPro" - um assistente especializado em gerar código Terraform seguindo padrões empresariais específicos.

Objetivos do Assistant:​

  • Gerar código Terraform seguindo guidelines da empresa
  • Sugerir best practices de Infrastructure as Code
  • Revisar código existente e sugerir melhorias
  • Responder dúvidas específicas sobre AWS, Terraform e IaC

PASSO 1: CONFIGURAÇÃO INICIAL​

Acessando GPT Builder:​

  1. Acesse ChatGPT Plus

    • Vá para chatgpt.com
    • Clique em "Explore GPTs"
    • Selecione "Create a GPT"

  2. Nome e Descrição

    Nome: TerraformPro DevOps Assistant
    Descrição: Especialista em Terraform para infraestrutura AWS com foco em best practices e padrões empresariais

  3. Avatar/Ícone

    • Use DALL-E para criar ícone temático
    • Ou faça upload de logo da empresa

PASSO 2: CONFIGURAÇÃO DAS INSTRUCTIONS​

Template de Instructions:​

Você é TerraformPro, um especialista sênior em Terraform e AWS com foco em Infrastructure as Code para ambientes empresariais.

## SEU PAPEL:
- Especialista em Terraform, AWS e DevOps
- Focado em produção enterprise-grade
- Seguidor rigoroso de best practices
- Comunicação clara e didática

## PADRÕES OBRIGATÓRIOS:
- Sempre usar Terraform >= 1.0
- Provider AWS >= 4.0  
- Variáveis tipadas sempre
- Outputs descritivos
- Tags padronizadas: Environment, Project, Owner, CostCenter
- Versionamento de state remoto S3

## ESTRUTURA DE CÓDIGO:
- Um arquivo por recurso principal
- Variáveis em variables.tf
- Outputs em outputs.tf  
- Providers em providers.tf
- Main logic em main.tf

## GUIDELINES:
- Sempre explique o porquê das escolhas
- Inclua exemplos práticos
- Foque em segurança e custos
- Sugira alternativas quando relevante
- Use comentários descritivos

## FORMATO DE RESPOSTA:
1. Explicação breve do contexto
2. Código Terraform comentado
3. Comandos de validação/apply
4. Considerações de segurança
5. Próximos passos sugeridos

Sempre pergunte sobre contexto específico quando necessário: ambiente, região AWS, requisitos de segurança, orçamento.

PASSO 3: KNOWLEDGE BASE​

Documentos para Upload:​

1. Terraform Style Guide​

# Terraform Style Guide Empresa

## Naming Conventions:
- Resources: snake_case
- Variables: snake_case  
- Modules: kebab-case
- Tags: PascalCase

## Required Tags:
- Environment: dev/staging/prod
- Project: nome-do-projeto
- Owner: email-do-responsavel
- CostCenter: centro-de-custo

## Module Structure:
terraform-aws-{service}/
├── main.tf
├── variables.tf
├── outputs.tf
├── versions.tf
└── README.md

2. AWS Best Practices​

# AWS Best Practices Internas

## Security:
- Sempre usar IAM roles, nunca access keys
- Encryption at rest obrigatório
- VPC privadas por padrão
- Security groups restritivos

## Cost Optimization:
- Usar tags para cost allocation
- Implementar lifecycle policies
- Reserved instances para workloads estáveis
- Monitoring de custos automatizado

3. Module Templates​

  • VPC module padronizado
  • ECS module da empresa
  • RDS module com encryption
  • S3 module com policies

PASSO 4: CONFIGURAÇÃO DE CAPABILITIES​

Habilitações Recomendadas:​

✅ Web Browsing​

  • Acessar documentação oficial Terraform
  • Verificar últimas versões de providers
  • Buscar soluções para problemas específicos

✅ Code Interpreter​

  • Validar sintaxe Terraform
  • Analisar outputs de terraform plan
  • Processar logs de erro

❌ DALL-E (opcional)​

  • Gerar diagramas de arquitetura
  • Criar documentação visual

PASSO 5: TESTES E VALIDAÇÃO​

Cenários de Teste:​

Teste 1: Criação Básica​

Prompt: "Crie um módulo Terraform para VPC com 2 subnets públicas e 2 privadas"

Resposta Esperada:
- Código seguindo style guide
- Tags obrigatórias incluídas
- Documentação clara
- Validações de input

Teste 2: Review de Código​

Prompt: "Revise este código Terraform: [cole código]"

Resposta Esperada:
- Identificação de problemas
- Sugestões de melhoria
- Alertas de segurança
- Otimizações de custo

Teste 3: Troubleshooting​

Prompt: "Terraform apply falhou com este erro: [cole erro]"

Resposta Esperada:
- Análise da causa raiz
- Solução passo-a-passo
- Prevenção futura
- Links para documentação

PASSO 6: PUBLICAÇÃO E COMPARTILHAMENTO​

Configurações de Acesso:​

Privado (Recomendado para Empresa)​

  • Apenas você pode acessar
  • Ideal para testes iniciais
  • Dados empresariais protegidos

Compartilhado via Link​

  • Qualquer pessoa com link pode usar
  • Bom para equipe específica
  • Controle de acesso limitado

Público no GPT Store​

  • Qualquer usuário ChatGPT Plus pode encontrar
  • Não recomendado para dados empresariais
  • Bom para casos genéricos

Recomendação:​

Mantenha privado durante desenvolvimento, depois compartilhe via link apenas com equipe DevOps.

ITERAÇÃO E MELHORIA​

Processo de Evolução:​

1. Coleta de Feedback​

  • Monitore como equipe usa o GPT
  • Identifique gaps no conhecimento
  • Observe padrões de perguntas

2. Atualizações Regulares​

  • Adicione novos modules à knowledge base
  • Atualize guidelines conforme empresa evolui
  • Refine instructions baseado no uso

3. Métricas de Sucesso​

  • Tempo reduzido para criar código Terraform
  • Diminuição de erros em produção
  • Adoção pela equipe
  • Qualidade do código gerado

MELHORES PRÁTICAS​

✅ Do's:​

  1. Seja Específico nas Instructions

    • Defina contexto técnico claro
    • Inclua padrões obrigatórios
    • Especifique formato de resposta

  2. Curate Knowledge Base

    • Documentos atualizados
    • Exemplos reais da empresa
    • Best practices validadas

  3. Teste Extensively

    • Múltiplos cenários
    • Diferentes tipos de perguntas
    • Validação com equipe

  4. Iterate Based on Usage

    • Monitore feedback da equipe
    • Refine baseado em usage patterns
    • Mantenha documentação atualizada

❌ Don'ts:​

  1. Não Inclua Informações Sensíveis

    • Nunca credentials ou secrets
    • Evite IPs ou configurações específicas
    • Mantenha dados confidenciais fora

  2. Não Crie Instructions Vagas

    • Evite "seja útil"
    • Seja específico em comportamentos
    • Define claramente o escopo

  3. Não Ignore Segurança

    • Configure acesso apropriado
    • Revise knowledge base regularmente
    • Monitore uso para vazamentos
